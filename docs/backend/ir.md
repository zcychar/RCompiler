IR backend notes
================

- Aggregate returns (structs/arrays) are lowered with an sret hidden pointer parameter. Functions returning aggregates take the sret pointer as the first argument and return `void`; the caller allocates the destination slot, passes it as the first argument, and reloads the value if needed. Scalars and `unit` still return normally.
- Conservative SROA runs before Mem2Reg. It splits aggregate allocas into scalar/pointer leaf allocas when every derived pointer use is a constant-index GEP ending at a scalar leaf. Safe whole-aggregate load-to-store copies are expanded into scalar leaf loads at the original load point and scalar stores at each destination, so later writes to the source do not change the copied snapshot. Dynamic indices, escapes, non-store aggregate-value uses, unsupported leaves, and large split counts remain in memory.
- Phi nodes survive the IR optimization pipeline in SSA form (Mem2Reg inserts them, but they are **not** demoted back to memory). The instruction selector lowers each phi into register-to-register moves (`mv`) emitted on predecessor edges, using a parallel-copy sequentialization algorithm that breaks cycles with temporary virtual registers. This avoids the alloca/store/load pattern entirely, which is critical for the REIMU target where loads and stores carry a weight of 64. Critical edges are split when a predecessor branches to multiple successors that contain phi nodes, so that edge-specific moves execute on the correct path only.
- **Function inlining** runs before SROA and Mem2Reg. It builds a call graph, finds SCCs via Tarjan's algorithm to detect recursive/mutually-recursive functions (which are skipped), then processes functions bottom-up in reverse topological order. Each eligible call site is replaced by a clone of the callee's body: the caller block is split at the call, parameters are mapped to arguments, and returns become jumps to a continuation block (with phi merges when multiple return sites exist). Sret aggregates need no special handling — the sret pointer is mapped like any other argument. Bottom-up ordering gives full transitive inlining in a single pass.
- The default `--opt=on` optimization order is: inline, SROA, mem2reg, constant propagation, CFG simplification, DCE, conservative ADCE, CFG simplification, final DCE. `--no-opt` disables the whole IR optimization pipeline.
- Constant propagation folds scalar arithmetic, unary operations, comparisons, simple casts, same-constant phi nodes, and constant branches. It does not fold division/remainder by zero, calls, loads, stores, or pointer/memory effects.
- CFG simplification removes unreachable blocks, folds conditional branches with identical targets, and bypasses empty jump-only blocks when phi predecessor rewrites remain unambiguous.
- DCE removes unused side-effect-free IR instructions. Stores, calls, and terminators are preserved as observable roots. Conservative ADCE performs a whole-function mark/sweep from observable roots but intentionally does not yet rewrite non-constant control dependence.
